# Summary Writing Framework

## Opening Hook Patterns

| Pattern | Example | Best For |
|---------|---------|----------|
| Role + Years + Domain | "Principal engineer with 10+ years..." | Safe, universal |
| Problem-Solution | "The system needed to scale 5x..." | Startups, storytelling |
| Contrarian | "Most systems fail from complexity..." | Differentiation |
| Specialization | "Go/Kubernetes engineer..." | Targeted roles |
| Value Proposition | "I build systems that don't break..." | Personal branding |

## Metric Types to Include

1. **Scale:** 100K users, 40K stores, 160K users
2. **Performance:** 83x improvement, 500ms→59ms
3. **Cost:** 50% reduction, 10x fewer servers
4. **Time:** Months→weeks, 4 weeks→10 days
5. **Reliability:** 99.9% uptime, zero-downtime

## Length Guidelines

| Platform | Ideal Length |
|----------|--------------|
| LinkedIn Summary | 200-300 words |
| Resume Summary | 2-4 lines (50-80 words) |
| GitHub Bio | 1-2 sentences |
| Email Signature | 10-15 words |

# A/B Testing Strategy

## How to Test

1. **LinkedIn:** Update summary, measure profile views after 2 weeks
2. **Resume:** Send different versions to similar roles, track response rate
3. **Cold Outreach:** Use different summaries in intro emails, measure replies

## Metrics to Track

- Profile view increase %
- Response rate to applications
- Interview conversion rate
- Recruiter inbound messages

## Testing Pairs (Similar Length, Different Angle)

- **1 vs 4:** Technical vs Business focus
- **5 vs 9:** Specialist vs Generalist metrics
- **8 vs 10:** Startup casual vs Startup sophisticated
- **11 vs 18:** Remote vs International positioning
